OUTPUT STYLING:

  1. IC INSIGHTS PROFESSIONAL TEMPLATE
     icInsightsBuilder.ts generates styled .docx with navy/gold palette.
     Integrated into outputBuilder.ts with fallback to basic unstyled doc.

  2. DATASET D POST-PROCESSOR (format_dataset_d.py)
     Alternating rows, auto-filters, conditional formatting (blast radius,
     priority scores, hub tags, gate status, Absence Claims), claim ID
     hyperlinks back to Claim Register, branded README. Called via
     child_process.execFileSync with graceful degradation.

  3. OUTPUT BUILDER REWRITE (outputBuilder.ts)
     IC Insights uses styled builder. Dataset D post-processed.
     Files renamed: CIMScan_Diligence_File, CIMScan_Run_Summary.

PIPELINE FIX:

  4. 0-CLAIM DETECTION + AUTO-RETRY (PIPELINE_ERR_011)
     pipeline.ts: guard after claimCount, returns retryable abort.
     anthropic.ts: Pass 1 prompt hardened against empty claims.

EMAIL ROUTING:

  5. FROM-BASED ROUTING (ingest.ts rewrite)
     Sender email is sole routing key. Message-ID dedup. Full validation
     chain. Unknown sender rejection. Audit logging to inbound_emails.

  6. MULTI-USER ARCHITECTURE (users table)
     Multiple analysts per firm. RLS policies. deals.user_id linkage.

  7. SIGNUP/ONBOARDING API (auth.ts)
     GET /api/firms (searchable dropdown). POST /api/auth/signup with
     existing firm selection or new firm creation + website validation.
     Supabase Auth email verification.

  8. 600 PE FIRMS SEEDED
     806 rows deduplicated to 600 unique firms. Ingest addresses derived
     from domain prefix. Zero collisions.
